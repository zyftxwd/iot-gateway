# 系统架构

## 总体目标

IoT Gateway 的目标是做一个完整的工业通讯管理平台，而不是单纯的协议 Demo。系统需要覆盖设备接入、实时采集、点表管理、报警、工单、资料归档、报表、权限和后续移动端接口。

## 架构分层

```text
前端 Vue
  设备管理 / 点表 / 实时数据 / 报警 / 工单 / 资料卡 / 报表 / 权限 / 接口测试

后端 Spring Boot
  Controller
  Service
  Protocol Handler
  Mapper
  Database

协议采集层
  Modbus TCP
  MQTT
  OPC UA
  Siemens S7 占位
  外部协议定义

数据闭环
  采集数据 -> 实时值 -> 历史存储 -> 报警判断 -> 工单 -> 验收 -> 维修资料卡
```

## 核心模块

- 资产模块：项目、分组、设备、点位。
- 协议模块：每个协议声明自身能力、连接字段、点位字段、发现方式和写入能力。
- 采集模块：按设备周期调度，异常隔离，单设备失败不应阻塞全局采集。
- 数据模块：实时数据、历史数据、保留小数、存储策略、质量标识。
- 报警模块：阈值、防抖、恢复延迟、报警事件、WebSocket 推送。
- 工单模块：报警转工单、派单、接单、处理、验收、关闭、附件归档。
- 资料卡模块：验收后的维修资料沉淀，用于后续同类问题推荐。
- 报表模块：历史采集、报警统计、工单统计、导出和打印。
- 权限模块：账号、部门、角色、项目级权限。
- 授权模块：为系统授权状态、协议能力开关和设备数量限制预留。

## 后续可扩展方向

- 多采集节点：将采集任务拆到多个节点，后端负责调度和状态汇总。
- 微服务：采集、报警、工单、报表、权限拆分。
- 移动端 API：所有菜单、标题、卡片、字段、按钮权限由后端返回。
- 授权能力：机器码、许可证文件、在线授权服务。
- 报表模板：字段布局、打印样式、保存方案、客户模板。
