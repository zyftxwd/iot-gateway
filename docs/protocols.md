# 协议能力和插件边界

## 当前内置协议

| 协议 | 状态 | 发现点位 | 写入 | 点表模板 | 说明 |
| --- | --- | --- | --- | --- | --- |
| Modbus TCP | 可用 | 手动/模板 | 支持 | 支持 | 适合寄存器和线圈类设备 |
| MQTT | 可用 | 支持 | 支持 | JSON 更合适 | 适合网关上报、主题订阅和 JSON 载荷 |
| OPC UA | 可用 | 树状浏览 | 支持情况取决于节点 | 支持 | 适合 KepServer、PLC 网关和标准 OPC UA Server |
| Siemens S7 | 占位 | 未完成 | 未完成 | 未完成 | 后续需要引入稳定 S7 客户端库 |

## 协议能力模型

后端协议接口应返回：

- 协议编码
- 显示名称
- 是否启用
- 是否支持发现点位
- 是否支持树状浏览
- 是否支持写入
- 是否支持 Excel 模板
- 是否支持 JSON 导入
- 默认端口
- 设备连接字段
- 点位字段
- 高级字段

前端新增设备和点位表单不应写死协议字段，而应根据协议元数据动态渲染。

## 点表字段差异

Modbus 需要：

- 地址
- 功能码
- 从站号
- 数据类型
- 字节序
- 字序
- 倍率
- 小数位
- 单位
- 读写权限

MQTT 需要：

- 订阅主题
- JSON 路径
- 数据标识
- 数据类型
- 小数位
- 单位
- 写入主题
- 写入报文模板
- 写入确认字段

OPC UA 需要：

- NodeId
- BrowseName
- DisplayName
- 数据类型
- 命名空间
- 是否可写

## 外部协议定义

当前系统支持导入协议定义 JSON，用于让前端新增设备表单动态增加协议选项和字段。这类导入适合声明协议元数据，但不能替代真实采集驱动。

如果一个新协议需要真实通讯能力，仍需要实现后端 `IProtocolHandler`。

## 商业版保留建议

建议在商业版中保留：

- 私有协议插件
- 高级协议驱动
- 分布式采集节点
- 高级授权校验
- 客户专用导入模板
- 移动端专用接口配置
- 报表模板设计器
